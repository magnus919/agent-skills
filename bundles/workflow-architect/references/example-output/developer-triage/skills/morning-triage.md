# Morning Triage

This skill is part of the **developer-triage** bundle, generated by
workflow-architect on 2026-05-29.

## When to Use

Activate when the user's opening message references status checks, pending
items, overnight updates, or PR assignments. This typically fires during
the first session of the day or after a long break.

## What to Do

1. **Sweep open items.** Check what PRs are assigned to the user, what issues
   need attention, and any overnight updates. Present a concise summary of
   what's pending.

2. **Flag urgent items.** If anything is time-sensitive or blocking someone
   else, surface it first.

3. **Suggest priorities.** Based on what's open, recommend an order of work.
   Offer to slot items into the current session.

4. **Ask about blockers.** Inquire whether anything is stuck or waiting on
   the user.

## Transition Signals

This phase ends when the user either:
- Picks a task and shifts into deep work ("ok let's work on X")
- Says they need to review a specific PR first
- Signals they're done with intake ("that's it, nothing urgent")

## What to Tell the User

Be concise and structured. Use bullet points for open items. Lead with the
most important change since the user's last session. Don't ask open-ended
questions like "how was your night" — keep it task-focused.

Example:
```
"You have 3 open PRs assigned to you, 1 new issue filed since yesterday,
and no blockers. PR #142 from Alex has been waiting 2 days — want to
start with that?"
```
